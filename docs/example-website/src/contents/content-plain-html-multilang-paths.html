Paths correction

  • Paths are adapted if the target directory is not at the same level
  • Therefore we don't have to think too much about paths if the resulting files are not at the same levels (e.g. /index-fr.html and /en/index-en.html(
  • Absolute paths are however left as-is, as expected
  • As are external links

If there is no leading ../, it is assumed automatically as our constents/masters directories should not contain something else, and therefore accessing resources requires to traverse the parent directory.

In fact, 2 attributes trigger these corrections: src and href, and we can use single and double quotes, or even no quote.: Examples: src='../images/a.png', href="../images/a.png".
